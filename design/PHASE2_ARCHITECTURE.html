二期架构 · 增加 readonly / edit 类型与额外工具

一期只交付 core + guide；二期在同一套 core 上接入「只读数据助手」「可编辑助手」与更多工具。本页用颜色标出二期相对一期的增量。

一期已建（core） 二期新增 二期可选 / 触发式

0. 二期相对一期加了什么

维度一期 (core + guide)二期新增
类型guidereadonly（查数据/分析）、edit（改数据）
主源前端 UIOpenAPI / MCP（给工具）
工具知识检索 + 导航数据工具、写工具、MCP/外部 API；工具数破 30 时动态检索
权限可见性层（行级可选）per-row 行级裁剪（readonly/edit 必需）
effect二元 read_only/has_side_effect完整 effect 契约（class/security/runtime）+ 写双门
写无WriteProposal → 人工确认 → executor 复核执行；interrupt 式写另需 durable checkpoint
身份Principal完整链 CredentialContext（委托/多租户时）
同步UI-diff+ OpenAPI oasdiff

core 一期就支持：manifest、agent_core、Provider 接口、SSE 契约、工具治理、eval gate、注册/发布——二期只在其上挂新模块，不改 core。

4. chatbot 问答流程（二期）

flowchart TD
  classDef p1 fill:#d6eaf8,stroke:#2471a3,color:#1a5276;
  classDef p2 fill:#fdebd0,stroke:#ca6f1e,color:#7e5109,stroke-width:3px;

  A["用户消息 + deployment_id + 类型"]:::p1 --> B["锁版本, 建图, 按权限裁剪"]:::p1
  B --> C{"tool-calling 循环"}:::p1
  C -->|怎么用、解释| K["取知识卡片"]:::p1
  C -->|查数据| T["数据工具"]:::p2
  T --> DS["DataScope 行级裁剪"]:::p2
  DS --> C
  K --> C
  C -->|够答即停| ANS["组答, 字段溯源"]:::p1
  ANS --> DONE["SSE 返回"]:::p1

  C -.->|变更请求| W{"显式 write mode 且 effect 有副作用"}:::p2
  W -->|否| NUDGE["提示切换写入模式"]:::p2
  W -->|是| PROP["生成 WriteProposal"]:::p2
  PROP --> CONFIRM["人工确认"]:::p2
  CONFIRM --> EXEC["executor 复核 权限/版本/expiry/幂等 → 执行"]:::p2
  EXEC --> DONE

一期问答只有蓝色部分（知识卡片 + 可选只读）。二期新增：数据工具 + 行级裁剪（readonly），以及整条写分支——检测变更请求 → 必须显式 write mode 且工具 effect 有副作用 → 生成草案 → 人工确认 → executor 执行前再复核（权限/版本/expiry/幂等）。读路径仍只读、写永不自动。
